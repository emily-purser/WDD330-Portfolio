MDN Web Docs - Validating Forms

Validation done on browser is client side.

3 reasons to validate forms: we want the right data in the right format; it protects user data; it also protects us.

Textarea doesn't support the pattern attribute.

In order to control the look and feel of native error messages or deal with legacy browsers that don't support HTML's built-in form validation, you have to use JavaScript.

Constraint Validation API -> is supported in most browser and is used to make custom error messages.

MDN Web Docs - Using the Fetch API

Access-Control-Allow-Origin is not allowed to use a wildcard for requests with the credentials: 'include'.

You can create a new request object and pass that into fetch request. It takes the same parameters.

A request can only be used once, to use it again, you will have to clone it or pass the request to the new request object. Just make sure that the request or response hasn't been read or the request will fail.

Response Instances -> returned when fetch() promises are resolved.
